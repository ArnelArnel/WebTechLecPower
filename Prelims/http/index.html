Hypertext Transfer Protocol

Hypertext Transfer Protocol

  • It is a stateless communication protocol
  • It also do not keep information about clients in between requests
  • HTTP resources are identified using URI's (RFC 3986) or specifically HTTP URL's
  • Scheme (http or https)
  • Authority - user information/authentication credentials,host
  • Query - typically provides as key value (=) pairs with ampersand (&) separated between key value pairs.

HTTP Request Standard Methods

  • Get
    • Most commonly supported by all compliant general purpose serves
  • Head
    • Used to test validity and also identical to get
  • Post
    • Request is suppose to make changes processing of the representation enclosed in the message of the body request
  • Put
    • Create or replace the state pf the target source
  • Option
    • Allows the client to determine the option and requirements associated to the resource
  • Trace
    • Allows the client to see what is being received at the other end of the request chain and use that data for testing or diagnostic information.

Requests Message

  • Request Line
    • Begins with a method token then a Request-URI and protocol version then ending with CRLF.
  • Message Headers
  • Empty Line
  • Message Body

HTTP Response Messsages

  • Status Line
  • Messages Headers
  • Empty Line
  • Message Body

Method Properties

  • Safe Methods
    • Get,Head,Options,Trace
  • Idempotent Method
    • Get,Head,Options,Trace,Put,Delete

HTTP Status Codes

  • Informational
  • Success
  • Redirection
  • Client Error
  • Server Error

HTTP Message Headers

  • General Header Fields
    • These applies to both requests and responses although there are no relation to the data eventually transfered in the body.
  • Request Header Fields
    • Request header fields has more information regarding the client or the resources that are about to be fetched.
  • Response Header Fields
    • These headers contains additional info regarding the response, for example; the location or about name and version of the server itself.
  • Entity Header Fields
    • Entity header fields has more info about the content length, MIME-type, etc of the body of the entity.

HTTP Extensions Web Dav (RFC 4918)

  • Requests Methods
    • Request methods are defined to specify the action that is wanted to be performed for a given resource.
  • Status Codes
    • Are made to imply either a specific HTTP request has already been completed or not.

Content Negotiation

  • Accept-Encoding
    • Requests a header of which compression algorithm or specifically a content encoding that the client understands.
  • Accept-Language
    • Requests a header of which locale variant is liked better by the client and which languages the client will be able to comprehend.
  • Accept-charSet
    • Requests a header of which character set is understandable by the client.

Range Request

  • Accept Ranges
    • These are markers used to support the partial requests on the server.
  • Byte Range
    • The main purpose of byte range is basically to avoid the need to download files even if a small section is all that is required this is essential in terms of conserving bandwidth usage.

Authorization/Authentication

  • Proxy-authenticate
    • Allows the proxy server to transmit the request further by authenticating requests.
  • WWW-authenticate
    • Defines authentication method which is supposed to be used to get access on resources.

Content

  • Content Models
    • A content model is basically an explanation of the supposed contents of the element
  • Void Elements
    • Element which does not allow its content model to gain contents in any form under any given situation.

Attributes and Attribute Values

  • Global Attribute
    • Global attributes are attributes that are common in any elements of html. This means that they can be used in any element. But on the other side, there will be no effects on some elements
  • WAI-ARIA Attribute
    • Makes a way to create web contents and web applications more friendly for people with disabilities.
  • Character Reference
    • These are character encoding-independent mechanisms that is used in inputting any character.
  • Comments
    • Comment tags are not displayed on the browser but are used to put comments on the source code.
