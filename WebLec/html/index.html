Definition

  • HTML is the standard markup language for creating Web pages.
  • HTML stands for Hyper Text Markup Language

History

  • First introduce in later 1991 by Sir Tim Berners-Lee.
  • First hosted at CERN, and then at the IETF.
  • With the creation of W3C, HTML development changed venue again. They attempt at extending HTML in 1995 known as HTML 3.0 which is known as the HTML 3.2 that compelled in 1997 then HTML 4 followed in 1998 a year later when HTML 3.2 introduce.

Version History

  • HTML 1.0
    • It is release in 1991 and it is the first HTML release in the world and written by Sir Tim Berners-Lee. It is based off of Standardized General Markup Language that as generally reliable and easier to accept as a new language. And also it is a wide variety of different implementations of people all over the world.
  • HTML 2.0
    • Release in 1995 and become the first official set of standards for HTML. This includes the original specifications of HTML 1.0 but a few features are added.
    • Published as IETF RFC 1866 and added a RFC supplemental capabilities (RFC 1867, RFC 1942, RFC 1980, RFC 2070)
    • HTML Working Group defined HTML 2.0
  • HTML 3.2
    • Release in 1997 which supports tables, “Client-Side Image Maps” that allows user to click different areas of images that is reference to network resources.
    • HTML 3.2 start the browser wars between Internet Explorer and Netscape, and the codename WILBUR is approve
    • W3C Recommendation
  • HTML 4.0
    • Release in 1997 and published as a W3C Recommendation that offers three variations (Strict, Transitional, Frameset).
    • Cougar is the code name before it was becomes HTML 4.0
  • HTML 4.01
    • Published in 1999 and it becomes the W3C Recommendations.
    • It supports the Cascading Style Sheets (CSS), for easier way to control presentational elements.
    • In May 2000 HTML 4.01 was the ISO/IEC international standard.
  • XHTML
    • HTML and XML join forces to become Extensible Hyper Text Markup Language.
    • XHTML Versions
      • XHTML 1.0
        • In 1999 XHTML 1.0 was introduce and in 2000 it was officially adopted as W3C Recommendation.
        • The following versions corresponds in HTML 4.01
          • XHTML 1.0 Strict which is equivalent to strict HTML 4.01 that includes elements and attributes.
          • XHTML 1.0 Transitional which is equivalent to transitional HTML 4.01 that includes presentational elements such as center, font and strike.
          • XHTML 1.0 Frameset which is equivalent to frameset HTML 4.01 that allows for definition of frameset documents.
      • XHTML 2.0
        • This was created to represent documents for a wide range of World Wide Web.
        • New features are added like forms, frames, alt, single heading element, hyperlink and etc.
  • HTML 5
    • HTML 5
      • In 2008 Web Hyper Application Technology Working Group HTML5 was the first Public draft
      • 2012 Web Hyper Application Technology Working Group HTML5 Living Standard
      • 2014 W3C Recommendation
    • HTML 5.1
      • Introduced new elements, attributes, and features that extend HTML 5.0 such as setContext(), forceSpellCheck(), dialog elements and etc.
      • 2014 W3C Recommendation
    • HTML 5.2
      • W3C Recommendation
      • Makes the venerable plugin system obsolete. Extended by plugins downloaded code with high power over the users devices.
    • HTML Living Standard
      • “A living standard is always updated and improved. New features can be added, but old functionality cannot be removed. ” -WHATWG
    • Difference HTML4 and HTML5
      • HTML5 is always updated and improved that is always adding or introducing new elements to simplified codes.
      • HTM5L is the living standard while HTML4 is the old version of HTML but the other functionalities of HTML4 are retain in HTML5.
  • HTML Document Structure
    • HTML Document Structure typically has the file extension .html.
    • It has two main parts which is the head tag and the body tag.

      • <!DOCTYPE html> <!-This declaration tells the browser of what version of HTML page is written. ->
      <html> <!- Element that wraps all the other code of the content of document. This was called the HTML root element.->
      <head> <!- A container in which include the number of HTML elements that are not visible parts of the page such as CSS stylesheets and JavaScript files.->
      <title>Title of the page which is displayed in the tab of the browser. </title>
      </head>
      <body> <!-This holds the content of the document that is displayed in the browser. ->
      <h1>Element that define the most important heading of the content.</h1>
      <p>Element that defines the paragraph.</p>
      </body>
      </html >

  • HTML Document Components
    • Document Type Declarations
      • It is an instruction that associates a particular XML or SGML document with a Document Type Definition of particular version of HTML.
      • HTML 4.01 Document Type Declarations
        • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
          • Does not allow presentational markup, but the Cascading Style Sheets must me used instead.
        • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
          • Documents that uses presentational attributes, deprecated elements, and link targets.
        • < !DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd" >
          • Documents that use frames.
      • HTML 5 Document Type Declarations
        • <!DOCTYPE html>
          • A very short declaration which has a lack of reference to a DTD.

  • Content
    • Content Models
      • It is used to define the kind of content that can be found within an element in HTML.
      • Types of Content
        • Embedded Content
          • Content the imports another resource in document.
        • Flow Content
          • Typically contain text or embedded content.
        • Heading Content
          • Defines header section.
        • Interactive Content
          • It s used for individuals that can participate with the information they are consuming.
        • Metadata Content
          • It described by an XML-schema document.
        • Phrasing Content
          • Defines the text and the mark-up it contain.
        • Sectioning Content
          • Defines the content of headings and footer.
    • Void Elements
      • An element that only has a start tag and no end tag. And also it never allows it to have contents under any circumstances.
      • The following list of Void Elements
        1. <area>
        2. <base>
        3. <br>
        4. <col>
        5. <hr>
        6. <img>
        7. <input>
        8. <link>
        9. <meta>
  • Attributes and Attributes Valuse
    • Global Attributes
      • id - Specifies the unique identifier that must be unique among all the IDs
      • title - Displays in the tab of the browser
      • lang and xml:lang - Specifies the the primary language for the elements contents.
      • xml:base - Used on elements of XML Documents
      • dir - Specifies the element text directionally.
      • class - Specifies one or more class names for an HTML element.
      • style - Specifies the inline style for an element.
        • See the following links by W3C Global Attributes for more information.
    • Custom Non-Visible Data Attributes
      • It is an attribute that no namespace whose name starts with string "data-". This attribute are intended to store custom data to the page. In oder to use this attribute you must start the attribute with the "data-" before the name of your variable.
    • WAI-ARIA Attributes
      • WAI-ARIA stands for Web Accesibility Initiative - Accesible Rich Internet Applications.
      • This provides a framework for adding attributes to identify features for use interaction. And also it includes technologies to map controls.
      • The following are the correct use of ARIA.

          • <h1><button>HEADING 1 BUTTON</button> </h1>

          <label>Name: <input type="text"></label></h1>
